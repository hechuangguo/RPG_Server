# 系统 MySQL 创建 rpg_game 数据库

## 当前状态

- 系统服务：`mysqld` 已为 **active**，监听 `*:3306`
- 项目内临时实例：此前 `.mysql/` 下的 mysqld 已停止（避免端口冲突）
- 系统 root 认证：
  - **Unix socket 可用**：`mysql -u root -e "SELECT 1"` 成功（无密码）
  - **TCP 密码不可用**：`mysql -h 127.0.0.1 -u root -p'hcg@7565310'` 报 `Access denied`
- 应用库/用户：**尚未在系统实例上创建**（`rpg_table` 连接失败）
- 现有脚本已就绪：
  - [`tables/create_user_and_db.sql`](tables/create_user_and_db.sql) — 建库 + 用户 + 授权
  - [`tables/init.sql`](tables/init.sql) — 5 张表（`CharBase` 等，`` `binary` `` 保留字已修复）
  - [`tables/setup_database.sh`](tables/setup_database.sh) — 一键脚本（当前用 `-h 127.0.0.1`，会因 root TCP 密码失败）
- 配置已对齐：[`config/config.xml`](config/config.xml)、[`tables/database.credentials`](tables/database.credentials) 均为 `rpg_table` / `rpg_game`

## 执行步骤

### 1. 确认环境

```bash
systemctl is-active mysqld          # active
ss -tlnp | grep 3306              # 仅系统 mysqld，无 RPG/.mysql 进程
pgrep -af "RPG/.mysql" || true    # 应无输出
```

### 2. 执行建库建用户（socket 方式）

因系统 root 仅 socket 免密可用，**直接通过 socket 执行 SQL**（不用 `-h 127.0.0.1`）：

```bash
cd /home/hechuangguo/RPG
mysql -u root < tables/create_user_and_db.sql
mysql -u root < tables/init.sql
```

预期结果：
- 创建库 `rpg_game`（utf8mb4）
- 创建用户 `rpg_table@localhost` / `rpg_table@127.0.0.1`，密码 `rpg_table`
- 创建表：`CharBase`、`Relation`、`Friend`、`Mail`、`MapInfo`

### 3. 验证

```bash
mysql -u rpg_table -prpg_table -e "USE rpg_game; SHOW TABLES;"
mysql -u rpg_table -prpg_table -e "USE rpg_game; SELECT COUNT(*) FROM CharBase;"
```

验收标准：`SHOW TABLES` 列出 5 张表，`CharBase` 可读（行数 0 即可）。

### 4.（推荐）设置 root TCP 密码，便于后续脚本

使 [`tables/setup_database.sh`](tables/setup_database.sh) 未来可重复执行：

```bash
mysql -u root -e "
  ALTER USER 'root'@'localhost' IDENTIFIED BY 'hcg@7565310';
  CREATE USER IF NOT EXISTS 'root'@'127.0.0.1' IDENTIFIED BY 'hcg@7565310';
  ALTER USER 'root'@'127.0.0.1' IDENTIFIED BY 'hcg@7565310';
  GRANT ALL PRIVILEGES ON *.* TO 'root'@'127.0.0.1' WITH GRANT OPTION;
  FLUSH PRIVILEGES;
"
```

之后可用：

```bash
MYSQL_ROOT_PASSWORD='hcg@7565310' ./tables/setup_database.sh
```

### 5.（可选）导入测试数据

```bash
mysql -u rpg_table -prpg_table rpg_game < tables/seed_test_data.sql
```

## 风险与处理

| 现象 | 处理 |
|------|------|
| `Access denied for user 'root'` | 改用 `mysql -u root`（不带 `-h`）走 socket |
| 表已存在 | 脚本含 `IF NOT EXISTS`，可安全重复执行 |
| `setup_database.sh` 仍失败 | 先完成步骤 4 设置 root TCP 密码，或继续用 socket 手动执行 SQL |
| 端口再次被占用 | 确认无 `.mysql/` 实例后再 `systemctl restart mysqld` |
