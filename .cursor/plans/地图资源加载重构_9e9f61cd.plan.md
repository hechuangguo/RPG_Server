# 服务器地图资源加载重构计划

## 目标

- 几何/碰撞/出生点等**运行时真源**统一为 `Common/map/{mapId}`。
- 策划 Excel 源表统一为 `Common/DataDoc/*.xlsx` → 服务端打表产物 `database/*_config.lua`。
- **彻底删除** Server 侧旧地图路径（`maps/`、`maps/runtime`、`Map@file`、`.map` 兼容）及相关代码与文档。
- **彻底删除** Server 根目录 `DataDoc/` 路径及其文档引用，策划源表仅保留在 `Common/DataDoc/`。
- Scene 启动、移动校验、副本创建、登录默认出生点链路保持可用。
- 本次仅实施服务器相关代码与文档，不实现客户端拉表与打表逻辑。

## 双轨数据模型

| 轨道 | 路径 | 职责 | 消费方 |
|------|------|------|--------|
| **策划表** | `Common/DataDoc/map.xlsx` → `database/map_config.lua` | 地图目录：名称、类型、maxPlayer、enabled、version、addressableKey、默认出生点 | C++ `MapConfigLoader`、服务端 Lua |
| **共享几何** | `Common/map/{mapId}/*.json` | bounds、碰撞、spawns、建筑/NPC 布局等 | C++ `MapDataLoader`、Unity 客户端 |

```mermaid
flowchart TB
    mapXlsx[Common/DataDoc/map.xlsx]
    genData[gen_data.sh]
    mapConfigLua[database/map_config.lua]
    serverInfo[server_info.xml]
    mapConfigLoader[MapConfigLoader]
    sceneManager[SceneManager]
    mapLoader[MapDataLoader]
    commonMap[Common/map/{mapId}]

    mapXlsx --> genData --> mapConfigLua
    mapConfigLua --> mapConfigLoader
    serverInfo --> sceneManager
    mapConfigLoader --> sceneManager
    sceneManager --> mapLoader --> commonMap
```

**原则**：`map_config` 不替代 `Common/map` 的几何数据；启动时交叉校验 `mapId` 存在、`version` 一致、`enabled=true`。

## 二、服务器解析 map_config（C++）

### 2.1 新增 MapConfigLoader

新建 [`sdk/util/MapConfigLoader.h`](/home/hcg/RPG_Server/sdk/util/MapConfigLoader.h) / `.cpp`：

- 启动时用独立 `lua_State` `require("map_config")`（`package.path` 含 `database/`、`../database/`）。
- 解析为 `std::unordered_map<uint32_t, MapTableEntry>`，提供 `load()`、`find(mapId)`、`getDefaultSpawnMapId()`。

### 2.2 与 Scene 启动合并

- `server_info.xml` 的 `<Map>` 仅保留 `id`（可选 `maxPlayer` 覆盖策划表）。
- `createNormalScenesFromConfig`：`MapConfigLoader::load()` → 合并策划表 → `Scene::start()` → `MapDataLoader` 读 `Common/map/{mapId}`。

### 2.3 默认出生点

Record/Super 创角优先从 `map_config` 读 1001 的 `spawnX/Y/Z`，失败回退 [`LoginSpawnConfig.h`](/home/hcg/RPG_Server/sdk/util/LoginSpawnConfig.h) 常量。
