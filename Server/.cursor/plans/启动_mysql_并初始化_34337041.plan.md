# 启动 MySQL 并执行数据库初始化

## 当前状态

- 已安装：`mysql-server-8.0.45`（`mysql` / `mysqld` 在 `/usr/bin`、`/usr/sbin`）
- 服务状态：`mysqld` **inactive**（未运行）
- 初始化脚本：[`tables/setup_database.sh`](tables/setup_database.sh)（内部依次执行 [`create_user_and_db.sql`](tables/create_user_and_db.sql) + [`init.sql`](tables/init.sql)）
- 应用账号配置：[`tables/database.credentials`](tables/database.credentials)、[`config/config.xml`](config/config.xml) 已为 `rpg_table` / `rpg_game`

## 执行步骤

```mermaid
flowchart LR
    A[systemctl start mysqld] --> B[验证服务 active]
    B --> C[setup_database.sh]
    C --> D[create_user_and_db.sql]
    D --> E[init.sql]
    E --> F[rpg_table 连接验证 SHOW TABLES]
```

### 1. 启动 MySQL 服务

```bash
sudo systemctl start mysqld
sudo systemctl is-active mysqld   # 期望输出 active
```

若启动失败，查看 `sudo journalctl -u mysqld -n 30` 定位原因（端口占用、数据目录权限等）。

### 2. 执行一键初始化脚本

在项目根目录 `/home/hechuangguo/RPG` 执行（root 密码通过环境变量传入，**单引号**包裹以避免 `@` 被 shell 解析）：

```bash
cd /home/hechuangguo/RPG
MYSQL_ROOT_PASSWORD='hcg@7565310' ./tables/setup_database.sh
```

脚本将自动完成：

| 步骤 | 动作 |
|------|------|
| 1/3 | 创建库 `rpg_game`，用户 `rpg_table@localhost` / `127.0.0.1`，密码 `rpg_table`，授权 |
| 2/3 | 执行 `init.sql` 建表（`CharBase`、`Relation`、`Friend`、`Mail`、`MapInfo`） |
| 3/3 | 以 `rpg_table` 连接并 `SHOW TABLES` 验证 |

### 3. 验收标准

- `systemctl is-active mysqld` 为 `active`
- 脚本输出 `完成` 且无报错
- `SHOW TABLES` 列出 5 张表
- 可选：`mysql -u rpg_table -prpg_table -e "USE rpg_game; SELECT COUNT(*) FROM CharBase;"` 确认可读

### 4. 安全说明

- root 密码仅用于本次 `MYSQL_ROOT_PASSWORD` 环境变量，**不会写入**仓库文件或 git
- 若需测试数据，可另行执行：`mysql -u rpg_table -prpg_table rpg_game < tables/seed_test_data.sql`（本次计划不包含，除非你要求）

## 权限要求

- `sudo`：启动 `mysqld`（你已确认可用）
- 网络/本地 socket：连接 `127.0.0.1:3306`

## 失败时的处理

| 现象 | 处理 |
|------|------|
| `Can't connect to local MySQL server` | 确认 mysqld 已 active |
| `Access denied for user 'root'` | 核对 root 密码是否正确 |
| `CREATE USER` / `GRANT` 失败 | 用 root 手动执行 `create_user_and_db.sql` 后重跑脚本 |
| 表已存在 | 脚本使用 `IF NOT EXISTS`，可安全重复执行 |
