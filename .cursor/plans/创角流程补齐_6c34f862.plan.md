# 角色创建完整流程补齐计划

## 现状结论
当前仓库已具备基础创角链路：客户端 `C2S_CREATE_USER_REQ` → Gateway `OnCreateUser` → Record `createCharacter` → Gateway `S2C_CREATE_USER_RSP` 并刷新 `S2C_USER_LIST`（已连通）。

需补齐的主要缺口是“完整性”而非“从零实现”：
- 创角前后状态与归属校验需要与选角幂等流程严格对齐。
- 错误码/文案与协议文档需统一，避免客户端处理分叉。
- 创角链路的观测字段需与登录链路统一（txn/accid/userId/zone）。

## 执行步骤
1. 对齐创角错误码与客户端回包语义，确保 `code/msg` 可被客户端稳定映射（包含重名、上限、非法名、系统失败）。
2. 强化 Gateway 创角前置条件（仅 `ACCOUNT_OK`、必须已鉴权 accid、zone/gameType 一致）以及创角后会话角色集合更新。
3. 强化 Record 创角边界（长度/字符校验、每账号每区上限、唯一索引冲突分流、默认出生点写入）并补齐结构化日志字段。
4. 校正文档协议表与链路时序，保证 `C2S_CREATE_USER_REQ`/`S2C_CREATE_USER_RSP`、`REC_CREATE_CHARACTER_*` 与实现一致。
5. 构建并执行联调用例：空角色创角成功、第四角色失败、重名失败、创角后列表即时刷新、随后选角进图成功。

## 验收标准
- 创角请求在 `ACCOUNT_OK` 状态下稳定成功；返回 `userID` 非 0。
- 创角成功后同连接可立即收到刷新后的角色列表且可选角进入。
- 失败路径错误码稳定可复现（重名/上限/非法名/系统错误）。
- 日志可按 `txnId/accid/userId/zoneId/code` 检索同一事务。
