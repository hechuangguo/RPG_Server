# 七服务器单例改造计划

## 目标

将以下 7 个服务器改为单例（统一复用 [`sdk/util/Singleton.h`](/home/hcg/RPG/sdk/util/Singleton.h) 的 `LazySingleton`），与已完成的 `SceneServer` 模式保持一致：

- `SuperServer`、`ZoneServer`、`RecordServer`、`LoggerServer`、`GatewayServer`、`AOIServer`、`GlobalServer`

## 参照模式（已存在）

[`SceneServer.h`](/home/hcg/RPG/SceneServer/SceneServer.h) 当前写法即为目标范式：

```cpp
class SceneServer : public INetCallback, public LazySingleton<SceneServer>
{
public:
    friend class LazySingleton<SceneServer>;
    static SceneServer* Instance() { return &LazySingleton<SceneServer>::Instance(); }
private:
    SceneServer();   // 私有构造
public:
    ~SceneServer() = default;
```

入口 [`SceneServer/main.cpp`](/home/hcg/RPG/SceneServer/main.cpp) 用 `auto* server = SceneServer::Instance();` 获取。

## 每个服务器统一改动

对每个 `XxxServer`（共 7 个）：

### 1) 头文件 `XxxServer.h`

- 增加 `#include "../sdk/util/Singleton.h"`（若未包含）。
- 类声明加基类：`: public INetCallback, public LazySingleton<XxxServer>`。
- public 区加：
  - `friend class LazySingleton<XxxServer>;`
  - `static XxxServer* Instance() { return &LazySingleton<XxxServer>::Instance(); }`
- 将构造函数移到 `private:`（声明保持不变，仅可见性变化；`friend` 授权基类构造）。
- 析构保持 public（如 `RecordServer` 的 `~RecordServer` 关闭 MySQL，行为不变）。

### 2) 入口 `XxxServer/main.cpp`

- 将 `XxxServer server;` 改为 `auto* server = XxxServer::Instance();`。
- 后续 `server.Init(...)` / `server.Run()` 改为 `server->Init(...)` / `server->Run()`。
- 启动参数、日志路径、信号处理逻辑保持不变。

涉及入口文件：
- [`SuperServer/main.cpp`](/home/hcg/RPG/SuperServer/main.cpp)（`Init(cfg.superIP, superPort)`）
- [`ZoneServer/main.cpp`](/home/hcg/RPG/ZoneServer/main.cpp)（`Init("0.0.0.0", zonePort)`）
- [`RecordServer/main.cpp`](/home/hcg/RPG/RecordServer/main.cpp)
- [`LoggerServer/main.cpp`](/home/hcg/RPG/LoggerServer/main.cpp)（多参 Init + logDir）
- [`GatewayServer/main.cpp`](/home/hcg/RPG/GatewayServer/main.cpp)（clientPort/innerPort）
- [`AOIServer/main.cpp`](/home/hcg/RPG/AOIServer/main.cpp)（多参 Init）
- [`GlobalServer/main.cpp`](/home/hcg/RPG/GlobalServer/main.cpp)（`Init("0.0.0.0", globalPort)`）

## 注意点

- 构造函数定义在各自 `.cpp`，私有化后由 `friend LazySingleton<T>` 访问，编译无碍。
- `SuperServer` 构造为头内 inline（`SuperServer() : m_server(this) {}`），同样移入 private 即可。
- 改前先 grep 确认这 7 个类无其它外部构造点或 `Instance()` 误用，避免遗漏。
- `GlobalServer` 构造声明在头、定义在 `GlobalServer.cpp`，私有化后由 `friend` 授权，无障碍。
- 保持 `Instance()` 返回裸指针，与 `SceneServer` 调用风格统一。
